# file-manager

## 精神

**文件是知识的容器。管理好文件，知识才不会散落。**

每次写入前先想：应该放哪里？每次读取前先想：去哪里找？

## 存入规则（按类型）

### 新学到的6课题内容

**存入**：`1-数据库/6课题数据库/对应课题.md`

**格式**：
```markdown
### YYYY-MM-DD
- **来源**：[来源名](URL)
- **内容**：[一句话描述]
- **参考**：[链接]
```

**同时更新**：`2-知识库/6课题知识库.md`（汇总视图）

---

### 日报

**存入**：`1-数据库/日报存档/YYYY/M月/YYYY-MM-DD-日报-关键词.md`

**同时更新**：`2-知识库/日报索引.md`（加一行索引）

---

### 人物传记

**存入**：`1-数据库/人物传记存档/YYYY-MM-DD-人物名.md`

**同时更新**：
- `1-数据库/人物传记存档/人物传记索引.md`（加一行）
- 对应课题的 `6课题数据库/对应课题.md`（标注关联）

---

### 公众号文章

**存入**：`4-公众号/故事分享/YYYY-MM-DD-公众号-标题.md`

---

## 调用规则（读取）

### 找某天日报
```
1-数据库/日报存档/YYYY/M月/  → 找对应日期文件
2-知识库/日报索引.md        → 快速定位
```

### 找某个课题的积累
```
1-数据库/6课题数据库/对应课题.md
2-知识库/6课题知识库.md（汇总视图）
```

### 找某人物传记
```
1-数据库/人物传记存档/人物传记索引.md  → 找标题
1-数据库/人物传记存档/YYYY-MM-DD-人物名.md  → 读内容
```

## 更新规则

### 更新现有内容

**原则**：不覆盖原内容，只追加。

- 在文件末尾添加新记录（带日期）
- 不删除旧内容
- 不修改已归档的日报内容

### 更新索引文件

**日报索引**：`2-知识库/日报索引.md`
- 每次写日报后，在索引末尾加一行

**6课题知识库**：`2-知识库/6课题知识库.md`
- 每月末更新汇总视图
- 或当某个课题有重大突破时更新

## 入库检查清单

每次存入前问自己：

- [ ] 这个内容属于哪个课题？（或不属于6课题）
- [ ] 应该放在哪个文件夹？
- [ ] 格式对吗？（日期+来源+内容）
- [ ] 索引需要更新吗？

```bash
grep -r "健康\|睡眠" "1-数据库/6课题数据库/" --include="*.md"
```

---

## 注意事项

1. **不要复制内容到多个地方** — 只放在一个位置，用引用
2. **不要删除旧内容** — 知识是积累
3. **索引要及时更新** — 没有索引，内容会丢失
4. **命名要规范** — `YYYY-MM-DD-关键词.md`
