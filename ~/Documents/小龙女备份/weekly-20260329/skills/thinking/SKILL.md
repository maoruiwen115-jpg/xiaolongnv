# thinking

## 精神

**思考是主动的、有根据的行为，不是被动产生的念头。**

遇到问题时：先问元认知三问 → 再用四步工具拆解 → 最后做决策和行动。

---

## ⚡ 动作前思考流程（每次动作前必做）

**遇到任何任务时，按以下顺序思考：**

```
① 这个任务涉及文件操作吗？
   │ 是 → 先读 file-manager skill
   │ 否 → 继续
   │
② 有没有现有的相关 skill？
   │ 有 → 调用相关 skill（如 biography-research、daily-journal）
   │ 否 → 用思考模型判断
   │
③ 用四步思考模型：
   │ ① 定义 → 任务的本质是什么？
   │ ② 分类 → 有哪些可能的方法？
   │ ③ 比较 → 哪个方法最优？
   │ ④ 因果 → 这个选择的后果是什么？
   │
④ 做决策 → 执行
```

**强制规则：**
- 不跳步骤：该搜索就搜索，该更新数据库就更新
- 不硬编码路径：用常量（DB_DAILY/VAULT）
- 不凭空估算：天气/位置/天数都用实际数据

**示例场景：**

| 场景 | 判断 | 行动 |
|------|------|------|
| "帮我生成日报" | 涉及文件操作 ✅ | 先读 file-manager skill |
| "同步到飞书" | 有 feishu-sync skill ✅ | 直接调用 feishu-sync |
| "帮我找个信息" | 有 unified-search skill ✅ | 调用 unified-search |
| "帮我分析这个问题" | 无相关 skill ✅ | 用定义-分类-比较-因果分析 |

**核心原则：先思考，再动作。不要盲目执行。**

## 元认知三问（分析任何问题前必做）

1. **我究竟是怎么知道的？** → 信息来源是什么？可靠吗？
2. **我这样想有没有可能是错的？** → 反例存在吗？
3. **其他可能性吗？** → 第三选择存在吗？

---

## 四步工具（第一部分：基础）

### 1. 定义（Definition）

**概念 = 必要 + 准确的描述**

要确保：
- **必要**：这个概念不可或缺
- **准确**：描述精准，没有歧义

**定义的动作**：
- 这个词指的是什么？（本质）
- 它不是什么？（边界）
- 什么时候适用/不适用？
- 它是必要的吗？（不可或缺吗）

通过学习和实践来不断更新和丰富大脑中的**概念表**。

**示例**：
- "飞书同步"的定义 = 通过API将markdown内容插入飞书文档（必要+准确）
- "焦虑"的定义 = 对未来不确定事件的过度担忧（必要+准确）

---

### 2. 分类（Classification）

**良好分类 = 合理 + 完整（MECE原则）**

| 标准 | 含义 |
|------|------|
| **合理** | 各类之间不重叠（mutually exclusive） |
| **完整** | 所有情况都覆盖（collectively exhaustive） |

**分类的动作**：
- 用什么维度切分？
- 各类之间有没有重叠？
- 有没有遗漏的情况？
- 分类标准是什么？

**示例**：
- 飞书同步失败 → 技术原因（token/rate limit）vs 内容原因（格式/编码）
  - 合理：两者不重叠 ✅
  - 完整：覆盖主要两类 ✅
- 时间管理 → 表面是时间分配，真正问题是注意力管理

---

### 3. 比较（Comparison）

**两个前提：同一范畴 + 同一属性**

| 前提 | 含义 |
|------|------|
| 同一范畴 | 同一领域/类型的对象才能比较 |
| 同一属性 | 用同一个维度比较 |

**判断依据：**

```
判断依据 = 主观 + 客观
优先使用客观依据
```

当存在多个客观判断依据时：
- 根据重要性分配权重
- 为每个依据打分

**比较的动作**：
1. 确定比较的对象是否在同一范畴
2. 确定比较的属性是否同一
3. 区分判断依据是主观还是客观（优先客观）
4. 多标准时：分配权重 + 打分

**示例**：
- "AI工具哪个好？" → 先定标准：速度/成本/效果 各占多少权重
- "两个方案选哪个？" → 给每个标准打分，综合最高胜出

---

### 4. 因果（Causation）

**四步检验法：**

```
开始
  │
① A的确是B的原因吗？
  │ 否 → 结束（不是因果）
  │ 是 → 继续
  │
② A是B唯一的原因吗？
  │ 是 → 结束（单一因果）
  │ 否 → 继续
  │
③ B的原因还有哪些？
  │
④ 最重要的原因是什么？
  │
结束
```

**因果的动作**：
- 不只看单一因素
- 问：还有其他因素吗？
- 找到最重要的那个

**常见错误**：
- 把关联当因果（A和B相关 ≠ A导致B）
- 倒置因果（B发生了 → 以为是A导致的）
- 忽略其他因素

## 决策框架（第二部分：应用）

**通过定义、分类、比较和因果的组合应用，可以构建一个简单而实用的思考框架。**

### 做决策的标准流程

```
① 定义 → 我真正想要的是什么？我的兴趣和长处是什么？
② 分类 → 对可能的选项进行分类（合理+完整）
③ 比较 → 在分类基础上比较，挑选最符合需求的选项
         （同一范畴+同一属性，客观优先，多标准权重+打分）
④ 因果 → 评估不同选择对未来可能产生的影响
         （四步检验：A是原因吗？唯一吗？还有哪些？最重要的是？）
```

### 示例：职业选择

```
① 定义：我的兴趣是什么？我的长处是什么？
② 分类：对可能的职业领域进行分类
③ 比较：在分类基础上比较，定标准+权重+打分
④ 因果：评估选择对未来的多维影响
```

## 进阶：关系（第三部分）

### 四种关系

| 关系 | 含义 | 辨别方法 |
|------|------|---------|
| **因果** | A导致B | 四步检验 |
| **相关** | A和B同时发生 | 有关联但不是因果 |
| **无关** | A和B没有联系 | |
| **互为因果** | A导致B，B也导致A | 最容易忽略 |

### 关系分类图

```
分类关系：
  从属关系
  同属关系：相交（与/或/非）、不相交
比较关系：
  约等于、大于、小于
因果分析：
  因果、相关、无关
```

### 因果与维度的三层世界

| 世界 | 特点 |
|------|------|
| **一维因果世界** | 因果/相关/无关，时间只是干扰因素 |
| **二维因果网络世界** | 互为因果，时间必要且不可忽略 |
| **三维因果系统世界** | 多维、动态的复杂系统 |

**警惕**：因果不是一维的，要考虑多维度和时间因素。

## 进阶：未知（第四部分）

### 人类思考的核心目标

> 探索未知，尤其是**预测未来**。

### 理解未知的三种工具

| 工具 | 适用场景 |
|------|---------|
| **形式逻辑**（三段论） | 确定性的推理（大前提→小前提→结论） |
| **统计概率** | 不确定性的量化分析 |
| **非形式逻辑/谬误** | 识别日常思维中的逻辑错误 |

### 三段论示例

```
大前提：人都会死的
小前提：苏格拉底是人
结论：苏格拉底会死的
```

```
大前提：大多数政客都会说谎
小前提：丘吉尔是个政客
结论：这一次丘吉尔说谎了吗？（概率问题，不是确定答案）
```

### 思维工具进化史

```
公元前4世纪 → 形式逻辑（三段论）
17世纪     → 统计概率
1970s      → 非形式逻辑/谬误列表
最近20年   → 贝叶斯推理（全面应用）
```

## 进阶：曙光（第五部分）

### AI的终极算法：贝叶斯推理

> 某件事发生的概率 = 随着新证据的出现而改变

**贝叶斯思维**：
- 先有一个先验判断（基于已有知识）
- 遇到新证据 → 更新判断
- 新判断成为新的先验 → 继续更新
- 如此循环，越来越准确

**应用到思考**：
- 不要固执于最初判断
- 新证据来了 → 更新
- 保持开放的思维概率

## 常见思维陷阱（进阶）

| 陷阱 | 真相 |
|------|------|
| 把互为因果当单向因果 | 最容易忽略 |
| 用一维思维看多维问题 | 因果是网络，不是直线 |
| 忽视时间因素 | 时间在因果中不可忽略 |
| 固执于最初判断 | 要用贝叶斯更新 |

## 返璞：简单才是终极复杂（第四部分）

### 类比

> 类比帮助我们理解复杂概念：通过将未知与已知建立联系，促进学习和理解。

```
未知 ≈ 已知
```

遇到新问题 → 找已知的类比 → 更容易理解。

---

### 发展：复杂性科学

```
极致简单 × 无限迭代 = 极度复杂 / 意外精彩
```

**Langton's Ant 例子**：
- 只有两条规则：白格→涂黑右转 / 黑格→涂白左转
- 但10000步后，产生惊人的复杂路径
- **简单规则 + 大量迭代 = 不可想象的复杂**

**类比到飞轮**：
- 飞轮规则很简单（输入→思考→判断→输出）
- 每天迭代
- 一年后，产生不可想象的成长

---

### 简单（简单 × 迭代）

```
什么更重要 → 多次迭代 → 什么最重要 → 价值观
```
