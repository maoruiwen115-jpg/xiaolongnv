# flywheel-engine

## 精神

**飞轮不是流程，是习惯。**

每天飞轮转一圈，我变强一点点。不是追完美，是每天比昨天好1%。

---

## 飞轮全景图

```
输入 ──→ 思考加工 ──→ 判断/分支 ──→ 循环迭代 ──→ 输出
             ↑                                    │
             └────────── 记忆更新 ───────────────┘
```

### 飞轮三要素

| 要素 | 内容 |
|------|------|
| **输入** | 用户对话 / 主动搜索 / 外部信息 / 新skill / 书籍 / 新闻 |
| **处理** | 判断（重要吗）→ 分支（用哪个skill）→ 循环（没处理好继续） |
| **输出** | 日报 / 决策 / skill更新 / 记忆 / 复盘 / 公众号 |

---


### 飞轮转动原则：能并联的都要并联

```
串联：哪些步骤必须按顺序执行
并联：哪些步骤可以同时进行

核心：能并联的都要并联，节省时间
```

**飞轮中的并联例子**：
- 早晨：输入（搜集信息）和思考（分析）可以并联
- 日报：内容生成和飞书同步可以并联
- 晚上：复盘和记忆更新可以并联

## 飞轮转动流程

### 早晨启动（7:40）

```
1. 读取今日context（memory-updater）
2. 问自己：今天飞轮转3件事是什么？
3. 启动飞轮，开始转动
```

### 白天运转

每遇到一件事：

```
① 判断：这个值得处理吗？
   → 不重要 → 跳过
   → 重要 → 继续

② 分支：用哪个skill处理？
   → 有对应skill → 调用
   → 没有 → 思考需要新skill吗？
           → 需要 → 创建/学习新skill
           → 不需要 → 直接处理

③ 循环：处理完了吗？
   → 没处理好 → 继续
   → 好了 → 问：有没有更好的方法？

④ 迭代：每次被纠正/犯错
   → 记下来
   → 写进memory/
   → 更新相关skill
```

### 晚上复盘（21:30-22:00）

```
1. 今天飞轮转得怎么样？
2. 学到了什么？（输入增加了）
3. 思考更深了吗？（判断更准了）
4. skill更强了吗？（工具更顺了）
5. 明天怎么转得更好？
```

---

## 飞轮检验标准

每次循环后问自己：

| 问题 | 如果是"是" |
|------|-----------|
| 我学到了什么？ | 输入增加了 |
| 我思考得更深了吗？ | 判断更准了 |
| 我能处理更多事情了吗？ | 分支更快了 |
| 我的skill更强了吗？ | 工具更顺了 |

**任何一个答案是"是" → 飞轮转成功了**

## 飞轮与日报的关系

日报是飞轮的**产物**，不是飞轮本身。

| | 飞轮 | 日报 |
|--|------|------|
| 定义 | 个人成长系统 | 每天的输出结果 |
| 衡量标准 | 我变强了吗 | 日报出了吗 |
| 频率 | 每时每刻 | 每天一次 |
| 核心价值 | 能力增长 | 记录成长 |

## skill如何联动

```
flywheel-engine（总控）
    │
    ├── just-do-it（行动触发：不拖延，立即做）
    ├── thinking（思考加工：定义-分类-比较-因果 四步）
    ├── find（判断输入）
    ├── web-search-ex（获取输入）
    ├── memory-updater（记忆更新）
    ├── self-improving-agent（迭代）
    ├── daily-review（复盘）
    └── [具体输出skill]
            ├── daily-journal（日报）
            ├── article-writer（公众号）
            └── [其他作品skill]
```

**just-do-it 是飞轮的行动触发器**：每次想说"等明天"，立即触发 just-do-it，让飞轮不停转。
