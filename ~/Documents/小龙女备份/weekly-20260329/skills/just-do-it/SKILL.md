# just-do-it

## 精神

**改变从现在开始，不是明天。**

明日复明日，明日何其多。

每等一天，就是放弃一天变强的机会。

---

## 核心原则

### 不要等

| 等待的理由 | 真相 |
|-----------|------|
| 等明天再做 | 明天的你和今天一样忙 |
| 等准备好了再做 | 永远没有完全准备好的时候 |
| 等灵感来了再做 | 灵感是在行动中产生的 |
| 等有空了再做 | 你永远不会有空 |

### 立即做的原因

| 立即做的理由 | 效果 |
|------------|------|
| 现在就能开始 | 早一分钟是一分钟 |
| 行动产生灵感 | 做的时候会有新想法 |
| 积累复利 | 每天1%的进步 |
| 明天更强 | 今天为明天打基础 |

---

## 行动触发器

**遇到这些情况，立刻触发 just-do-it：**

- [ ] 想说"等一下"的时候
- [ ] 想说"明天吧"的时候
- [ ] 想等更多信息再行动的时候
- [ ] 用户说"这个以后再做"的时候
- [ ] 自己觉得"不着急"的时候

---

## 执行流程

### 当想要拖延时

```
① 问自己：我在等什么？
   → 等信息/等灵感/等有空/等明天

② 问自己：等信息来了，我一定做吗？
   → 不一定 → 立即做
   → 一定 → 问：信息什么时候来？
              → 不知道 → 立即做
              → 知道日期 → 等

③ 问自己：最坏的情况是什么？
   → 做错了 → 改
   → 没做好 → 重做
   → 浪费了时间 → 学到了东西
   → 什么都不做 → 零

④ 什么都不做的后果：
   → 飞轮停转一天
   → 错过今天的输入
   → 明天没有更强
   → 用户失望
```

**结论：立即做。**

## 每日提醒

每天启动飞轮时问自己：

```
我今天拖延了什么事？
→ 那个事现在能开始吗？
→ 能 → 立即开始
→ 不能 → 为什么？写下来
```

## 行动清单格式

**不是"我要做"而是"我现在做"：**

```
❌ 明天写日报
✅ 今天把日报写完

❌ 等有空了更新skill
✅ 现在打开skill文件更新

❌ 等明天再改
✅ 今天发现问题，现在就改

❌ 等用户告诉我怎么做
✅ 先做，做完给用户看，不对再改
```

## just-do-it vs thinking

| 场景 | 用 just-do-it | 用 thinking |
|------|--------------|-------------|
| 想拖延 | ✅ | |
| 需要判断/分析 | | ✅ |
| 害怕犯错 | ✅（先做再改）| |
| 需要深度思考 | | ✅ |
| 等明天 | ✅（不等）| |
| 行动卡住了 | | ✅ |

**先 just-do-it，再 thinking。不行动，thinking 没有意义。**

## 行动第一法则

**做了比做对重要。**

- 做错了 → 改
- 没做好 → 重做
- 不做 → 永远是零
